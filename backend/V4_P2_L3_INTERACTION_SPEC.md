# V4 P2 · L3 — 交互规格

> 配套 L2 原型 `V4_P2_L2_CLICKABLE_PROTOTYPE.html`、L2.3 数据契约、L4 技术设计。
> 用户端**极简**（不暴露技能/参数/ffmpeg/manifest）；后台复杂度对普通用户隐藏。
> 所有接口带 `Authorization: Bearer <JWT>`；统一壳 `{code,message,data}`。

---

## 0. 两类视角
| 视角 | 可见 |
|----|----|
| 普通用户 / invite_admin | 选用途 → 上传 → 一句话 → preview → 确认生产单 → 裂变计划 preview → 确认 → 30 条 → 筛选/下载/收藏/反馈 |
| super_admin | 以上 + 技能库(只读运维视角) + 素材包管理 + 候选池审核 + QA 概览 + 全局存储 |

## 2. 控件状态机（按钮级）

### 2.1 「生成母视频（A台）」按钮
| 状态 | 条件 | 外观/动作 |
|----|----|----|
| 默认 | 已 preview 且确认费用 | 可点 → `POST /api/production-orders` |
| 禁用 | 未 preview | 灰 + tooltip「请先生成导演稿预览」 |
| 锁态 | compose 返回 4031 | 置灰 + 显示「生成通道维护中，暂不可用。」 |
| 余额不足 | 4029 | 弹「额度不足，请联系管理员充值」 |

### 2.2 「确认裂变（B台）」按钮（0 成本）
| 状态 | 条件 | 外观/动作 |
|----|----|----|
| 默认可用 | 合格源视频 ≥3（`duration_seconds>=30`） | 绿色「0 元/条」→ `execute` |
| 禁用 | 合格源 <3 | 灰 + tooltip「至少 3 个时长 30 秒以上的视频」 |
| 运行中 | 轮询 status=executing | 旋转 + 「裂变中 N/30」 |
| 完成 | done | 角标「N 条新」+ 滚动到结果区 |
| 含 QA fail | done 且有 fail | 「N 条新，M 条需重做」 |

### 2.3 视频卡片（裂变结果）
| 状态 | QA | 外观 | 操作 |
|----|----|----|----|
| 通过 | final_status=pass | 正常卡片 + 绿点 | 播放/下载/收藏/反馈 |
| 警告 | warn（相似/品牌/字幕） | 黄点 + tooltip 原因 | 同上（可下载） |
| 失败 | fail（pts/playable/duration） | 红点「需重做」 | 重做 / 删除（不建议下载） |

## 3. 关键异常/锁态文案（逐字）
| 场景 | code | 文案 |
|----|----|----|
| 未登录/过期 | 401/1001 | 跳登录页 |
| 权限不足（跨租户/非super_admin） | 403/1001 | 「该功能需要管理员权限」 |
| compose 熔断锁 | **4031** | 「生成通道维护中，暂不可用。」（A台生成按钮置灰） |
| 图片不可访问 | **2002** | 「图片无法被视频模型访问，请重新上传或等待处理完成。」 |
| B台源视频不足 | 2001 | 「请至少上传 3 个时长 30 秒以上的视频，才能稳定裂变。」 |
| 额度/试用不足 | 4029 | 「额度不足，请联系管理员充值」 |
| QA 不通过（单条） | — | 卡片标「需重做：{原因}」（如「PTS 异常 / 不可播放到结尾 / 重复」） |
| 反馈失败 | 2001 | 「反馈提交失败，请重试」（不影响播放/下载） |
| 埋点失败 | — | 静默忽略（fire-and-forget，不阻断） |

## 4. 裂变计划 preview 展示（让用户「看得见为什么 30 条」）
preview 返回 `fission_plan`，前端渲染：
- 顶部：「将生成 **30** 条，分 **6** 组」。
- 6 组卡片：组名（痛点前置/卖点前置/效果收束/品牌双定/同源前后/倒叙）+ center_idea + 每组 5 条。
- 每条可展开：由哪两/三段组成（segment_plan 的 role 序列）、是否加文字卡/转场/品牌定帧。
- **不展示**：skill_id、ffmpeg 参数、PTS、技能序列细节（这些是后台）。

---

## 5. 候选池 / 技能库 / 素材包（super_admin 视角）
- **候选池**：`GET /api/admin/knowledge-candidates`，pending 列表 + approve/reject；user/invite_admin 不渲染该区。
- **技能库**：`GET /api/skills` 只读列表（skill_id/name/category/risk/version/启用态）——**仅运维视角，不给普通用户参数**。
- **素材包**：`GET /api/asset-packs` + `POST /api/asset-packs`（建/改品牌包）；普通用户只在生成时「选择品牌包」，不编辑。
- **QA 概览**：按生产单聚合 pass/warn/fail 数（super_admin）。

## 7. 极简原则落地（用户永不看到的东西）
- 不看：skill_registry 参数、ffmpeg/ffprobe 命令、shot_map 时码、manifest JSON、PTS/MD5/相似度数值。
- 只看：场景选择、素材缩略图、导演稿（人话分镜）、估价、裂变计划（6 组 + 为什么 30 条）、结果卡片（绿/黄/红点）。
- super_admin 才进后台（技能/素材/候选池/QA），且仍以「列表 + 开关」呈现，不暴露裸参数。
