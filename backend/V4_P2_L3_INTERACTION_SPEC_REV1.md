# V4 P2 · L3（Rev1）— 交互规格

> Rev1（取代原 L3）。主修：时长口径短视频、**素材增强区（免费优先/付费外部跳转）**、QA 重做/partial_done、API 语义澄清、权限分层。
> 用户端极简；接口带 JWT；统一壳 `{code,message,data}`。

## 0. 两类视角
| 视角 | 可见 |
|----|----|
| user / invite_admin | 用途→上传(+素材增强)→一句话→preview→确认生产单→裂变计划 preview→确认→短视频结果→筛选/下载/收藏/反馈 |
| super_admin | 以上 + 技能库(只读) + **品牌素材包**管理 + 候选池 + QA 概览 + 全局存储 |

> 权限分层：**production_assets（本次上传）普通用户可传**；**brand_asset_pack（品牌包）super_admin/授权管理员**管理。前端两者不混。

## 2. 素材增强区（★ Rev1 新增）
上传区旁「素材使用方式」三选一（默认②）：
1. **仅使用我上传的素材**
2. **优先使用免费素材**（默认）— 系统从 free_stock_gateway(Pexels/Pixabay/Unsplash) 自动检索+评分推荐，0 成本。
3. **允许推荐付费素材** — 展开预算设置：
   - 单条素材预算：¥5 / ¥10 / ¥30 / 自定义
   - 本次生产总预算上限：¥50 / ¥100 / ¥300
   - 是否需人工确认每个付费素材（默认开）
   - 提示（逐字）：**「付费素材由你直接向素材平台购买，费用不经过本平台。购买后请下载素材并上传到本次生产任务。」**

**付费素材卡片**（展示，不代购）：素材预览 / 来源平台 / 价格或区间 / 授权提示 / **[官方购买]**(外部跳转) / **[购买后上传]**。
- 点 [官方购买] → 跳转 Adobe/Shutterstock/Getty/Storyblocks 官方页（external_redirect），用户自行扫码/刷卡支付。
- 用户下载后回 V4 [购买后上传] → 系统标 `paid_user_uploaded_asset`，弹**授权确认**：「我确认该素材由我本人购买或拥有合法使用权，并授权本次视频生产使用。」勾选后方可用于本单。

## 3. 裂变计划 preview 展示（让用户「看清为什么 30 条 + 将用哪些素材」）
preview 返回 `fission_plan`，渲染：
- 顶部：「将生成 **30** 条，分 **6** 组」+ 6 组卡（痛点前置/卖点前置/效果收束/品牌双定/同源前后/倒叙，每组 5 条）。
- **素材构成**（asset_summary）：
  ```
  将使用：上传素材 3 个 / 品牌素材 5 个 / 免费素材 8 个 / 付费素材 0 个
  预计素材成本：¥0
  ```
- 若有付费素材：「推荐付费素材 2 个，预计素材成本 ¥38（由你向素材平台支付，不经本平台）。是否确认使用？」用户确认（且已上传/已授权）后才进执行。
- **不展示** skill_id / ffmpeg 参数 / PTS / 技能序列（后台）。

## 4. 控件状态机
### 4.1「生成母视频(A台)」
| 状态 | 条件 | 动作 |
|----|----|----|
| 默认 | 已 preview + 确认费用 | `POST /api/production-orders` |
| 锁态 | 4031 | 置灰 +「生成通道维护中，暂不可用。」 |
| 余额不足 | 4029 | 「额度不足，请联系管理员充值」 |

### 4.2「确认裂变(B台)」（0 平台成本）
| 状态 | 条件 | 动作 |
|----|----|----|
| 默认可用 | 合格源≥3（`duration_seconds>=30`）且付费素材已确认/已上传 | 绿色「0 元/条」→ execute |
| 禁用 | 合格源<3 | 灰 + tooltip「至少 3 个时长 30 秒以上的视频」 |
| 待确认 | 有付费素材未确认 | 灰 + 「请先确认/上传付费素材」 |
| 运行中 | executing | 「裂变中 N/30」 |
| 完成 | done | 「N 条新」（含 partial：「N 条可用，M 条需重做」） |

### 4.3 视频卡片（短视频结果，QA）
| QA | 外观 | 操作 |
|----|----|----|
| pass | 绿点 | 播放/下载/收藏/反馈 |
| warn（相似/品牌/字幕/license soft） | 黄点 + tooltip | 同上（可下载，提示风险） |
| fail（pts/playable/duration/md5/license hard） | 红点「需重做」 | 重做/删除（**不建议下载**） |

## 5. 异常 / 锁态文案（逐字）
| 场景 | code | 文案 |
|----|----|----|
| 未登录/过期 | 401/1001 | 跳登录页 |
| 权限不足 | 403/1001 | 「该功能需要管理员权限」 |
| compose 锁 | **4031** | 「生成通道维护中，暂不可用。」 |
| 图片不可访问 | **2002** | 「图片无法被视频模型访问，请重新上传或等待处理完成。」 |
| 源视频不足 | 2001 | 「请至少上传 3 个时长 30 秒以上的视频，才能稳定裂变。」 |
| 额度不足 | 4029 | 「额度不足，请联系管理员充值」 |
| 付费素材未授权确认 | — | 「请先确认你已合法购买并授权该素材」 |
| QA fail（单条） | — | 「需重做：{PTS异常/不可播放到结尾/重复/授权缺失}」（自动重做≤2 次） |
| batch partial | — | 「{N} 条可用，{M} 条需重做」 |
| 反馈失败 | 2001 | 「反馈提交失败，请重试」（不阻断） |
| 埋点失败 | — | 静默忽略 |

## 6. QA 重做 / partial_done 交互（★ Rev1）
- 单条 fail → 后台自动重做（≤2 次），前端显示该条「重试中」。
- 仍失败 → 标「需重做」红点，不入可下载列表。
- batch 不因个别失败卡死：完成即展示「可用 N 条」，失败条单独列「需重做 M 条」可手动重试/删除。

---

## 7. super_admin 后台（仍以列表+开关，不暴露裸参数）
- 候选池：`/api/admin/knowledge-candidates`，approve/reject（user 不可见）。
- 技能库：`GET /api/skills` 只读（skill_id/category/engine/risk/version/启用态）——不给参数。
- **品牌素材包**：`GET/POST /api/asset-packs`（建/改）；普通用户只在生成时「选品牌包」，不编辑。
- 素材源：`GET /api/asset-sources`（免费/付费源启用态、授权策略）。
- QA 概览：按生产单 pass/warn/fail 聚合。

## 8. 极简原则（用户永不看到）
技能/参数/ffmpeg/PTS/MD5/manifest/相似度数值、素材授权台账细节、provider API。用户只看：用途、素材缩略图、素材增强三选项、导演稿(人话)、估价、裂变计划(6组+素材构成+成本)、结果(绿黄红点)。
