# V4 页面重构 P1 · 交互规格 (L3)

> 配套 L2 原型 `V4_P1_WIREFRAME_PROTOTYPE.html`、L4 技术设计 `V4_P1_TECHNICAL_DESIGN.md`。
> 颗粒度：**按钮级 / 状态级 / 异常级**。所有弹窗文案**逐字写出**，禁止「参考某某」。
> 真实后端依据：`claude/v4-staging` 分支。所有接口带 `Authorization: Bearer <JWT>`；统一返回 `{code,message,data}`。

---

## 0. 角色与全局

| 角色 | 来源 | 可见区域 |
|----|----|----|
| `user` | 登录默认 | 工作台 + 两个陈列面 + 自己租户存储栏 |
| `invite_admin` | super_admin 授权 | 同 user + 发码入口（本页不含发码 UI） |
| `super_admin` | bootstrap | 全部 + **回流候选池** + **全局存储栏(scope=global)** |

- 角色由登录返回的 `role` 决定；进入工作台先调 `GET /api/me` 确认 `role/is_admin`。
- 候选池区域、全局存储数据**仅 `super_admin` 渲染**；其余角色前端不渲染该 DOM（后端也会 403/scope=tenant 双重保护）。

## 1. 控件状态机（按钮级）

### 1.1 「A台 · 母视频」按钮
| 状态 | 条件 | 外观 | 可点击 | 点击后 |
|----|----|----|----|----|
| 默认 | 需求框非空 | 橙色 + 副标「⚠️ 会产生费用」 | ✅ | 弹「A台费用确认弹窗」 |
| 禁用 | 需求框为空 | 橙色半透明 + tooltip「请先描述需求」 | ❌ | — |
| 确认中 | 费用弹窗已弹 | — | — | 「确认生成」→ 调 `POST /api/a/generate`；「取消」→ 关闭 |
| 生成中 | 任务 pending/running | 橙色 + 旋转图标 + 文字「正在生成第 N 段/共 M 段」 | ❌ | 轮询 `GET /api/tasks/{id}` |
| 拼接中 | 多段已生成，concat 阶段 | 文字「正在拼接…」 | ❌ | 继续轮询 |
| 完成 | task.status=done | 橙色默认 | ✅ | 母视频陈列面新增卡片 + toast「母视频已生成」 |
| 失败 | task.status=failed | 橙色默认 | ✅ | toast 错误（见异常矩阵） |

> 备注：A台**会扣费**，进入确认弹窗前不发任何请求；确认后才请求。

### 1.2 「B台 · 裂变」按钮（P1 核心改动：自动选源 + 1:10）
| 状态 | 条件 | 外观 | 可点击 | 点击后 |
|----|----|----|----|----|
| 默认可用 | 本租户「时长≥30s 的源视频」≥3 个 | 绿色 + 副标「0 元/条」 | ✅ | 弹「裂变进行中弹窗」，调 `POST /api/b/batch-generate` |
| 禁用·数量不足 | 合格源视频 < 3 | 灰色 + tooltip「至少上传3个时长30秒以上的视频」 | ❌ | （若强行点）弹「裂变门槛弹窗」 |
| 禁用·时长不足 | 有视频但合格(≥30s)的 < 3 | 灰色 + tooltip「存在时长不足30秒的视频」 | ❌ | 弹「裂变门槛弹窗」 |
| 加载中 | 裂变请求已发出，等待 batch_id | 绿色 + 旋转图标 | ❌ | — |
| 运行中 | 轮询 batch.status=running | 绿色 + 旋转 | ❌ | 轮询 `GET /api/b/batch/{batch_id}` |
| 完成 | batch.status=done | 绿色 + 角标「N 条新」 | ✅ | 滚动到裂变陈列面并刷新 |
| 部分失败 | done 且 failed>0 | 绿色 + 角标 | ✅ | toast「部分裂变失败，已成功 N 条」（不阻断） |

> **P1 规则**：前端**不再要求勾选源视频**。点 B台后端**自动选取**本租户 `type=mother & storage_status=active & 时长≥30s` 的源视频，按 1:10 产出，30/40/50 封顶（见 L4）。

### 1.3 上传入口（📷图片 / 📎文件 / 🎬视频）
| 状态 | 条件 | 外观 | 行为 |
|----|----|----|----|
| 默认 | — | 虚线框可点 | 选择文件 |
| 校验中 | 选中文件后本地校验 | — | 大小/格式前端预校验（见异常矩阵 413/格式） |
| 上传中 | 请求中 | 进度条 | `POST /api/uploads/batch` |
| 成功 | 200 | 素材栏计数 +N，缩略图出现 | 视频额外进母视频陈列面 |
| 部分失败 | uploaded 与 failed 都有 | 失败文件标红 | toast「N 个文件上传失败」，成功的保留 |

> **无文本上传入口**（已删）。**无蓝色「上传素材」按钮**（已删），底部只剩 A台 + B台。

### 1.4 视频卡片（母/裂变共用）
| 状态 | 外观 | 操作 |
|----|----|----|
| active | 正常卡片 | 播放 / 下载 / 删除 (/ 反馈：仅裂变卡片) |
| 删除中 | 卡片置灰 + 旋转 | `DELETE /api/videos/{id}` 进行中 |
| 已删除 | 从默认列表移除 | 默认列表不显示（`include_expired=true` 才显示 deleted/expired） |
| 已过期 | 灰标「已过期」 | 仅在 `include_expired=true` 列表出现，文件已被清理 |

### 1.5 候选池条目（super_admin）
| 状态 | 外观 | 操作 |
|----|----|----|
| pending | 待审卡片 | 审核通过 / 驳回 |
| approved | 绿标「已通过」 | 只读（P1 不推送大库） |
| rejected | 灰标「已驳回」 | 只读 |

## 2. 异常处理矩阵（完整）

| 异常场景 | 前端行为 | 提示文案（逐字） | 是否阻断 |
|----|----|----|----|
| 网络断开 | 所有请求 catch → toast | `网络连接失败，请检查网络` | 阻断 |
| 接口超时(>30s) | retry 1 次 → 仍失败 toast | `请求超时，请稍后重试` | 阻断 |
| 401 未登录/令牌过期 | 清 token → 跳登录页 | （无 toast，直接跳转） | 阻断 |
| 403 权限不足 | toast | `该功能需要管理员权限` | 阻断 |
| 403 跨租户删除 | toast | `无权删除其他租户的视频` | 阻断 |
| 413/超 2G | 上传前本地拦截 | `文件超过 2G 限制` | 阻断 |
| 格式不支持 | 上传前本地拦截 | `不支持该文件格式` | 阻断 |
| A台余额不足 (4029) | 弹窗 | `余额不足，请联系管理员充值` | 阻断 |
| A台一句话超额 (2001) | toast | `一句话最多生成 10 条母视频，请拆分` | 阻断 |
| B台源视频不足/时长不足 (400/2001) | 弹「裂变门槛弹窗」 | 见 §3 弹窗文案 | 阻断（不发起裂变） |
| B台裂变失败 | toast + 保留已成功部分 | `部分裂变失败，已成功 N 条` | 不阻断 |
| `events/track` 失败 | 静默忽略（fire-and-forget） | 无 | 不阻断 |
| `feedback` 失败 | toast | `反馈提交失败，请重试` | 不阻断（不影响播放/下载） |
| 候选池审核失败 | toast | `操作失败，请重试` | 不阻断 |

## 4. 边界条件（完整）

| 边界 | 场景 | 行为 |
|----|----|----|
| 0 个视频 | 新注册无数据 | 两个陈列面显示空态；B台禁用 |
| 1 个视频 | 上传 1 个 | B台禁用，tooltip「至少上传3个时长30秒以上的视频」 |
| 2 个视频 | 上传 2 个 | B台禁用，同上 |
| 3 个视频但 1 个 <30s | 时长不足 | B台禁用，tooltip「存在时长不足30秒的视频」 |
| 正好 3 个且都 ≥30s | 刚满足 | B台可用，裂变 **30** 条 (3×10) |
| 4 个且都 ≥30s | — | 裂变 **40** 条 (4×10) |
| 5 个及以上且都 ≥30s | 充足 | 裂变 **50** 条 (封顶 50) |
| 单文件 >2G | 超限 | 上传前本地拦截 |
| 同时上传 10 个文件 | 批量上限 | 正常处理，进度条显示（每类 ≤10） |
| 上传中网络中断 | 异常 | 失败文件标红，成功文件保留 |
| A台一句话要求 >10 条 | 超额 | 后端 2001，toast 提示拆分 |

## 6. 关键交互流（端到端）

### 6.1 上传 → 陈列面
1. 点 🎬视频 → 选文件 → `POST /api/uploads/batch`。
2. 成功响应 `uploaded[].video_id` 非空 → 刷新 `GET /api/videos?type=mother&source_type=uploaded` → 卡片出现（标签「本地上传」）。
3. 解决「上传了不知道去哪用」：上传 video 即时进入母视频/源视频陈列面。

### 6.2 A台一键成片（文字+图片→多段15s→拼接→母视频）
1. 填需求（可选上传图片做参考）→ 点 A台 → 费用确认弹窗 → 确认。
2. `POST /api/a/generate`（或一句话 `POST /api/generate`）→ 返回 `task_id`。
3. 轮询 `GET /api/tasks/{task_id}`，展示「第 N 段/共 M 段」「正在拼接」。
4. `status=done` → 刷新母视频陈列面（标签「A台生成」）。

### 6.3 B台批量裂变（自动选源 + 1:10）
1. 点 B台 → 前端先本地判断合格源视频数（≥3 且 ≥30s），不足弹门槛弹窗。
2. 满足 → `POST /api/b/batch-generate {prompt, total_limit?}`（**不传 sources，后端自动选**）→ 返回 `batch_id, total`。
3. 弹「正在裂变」，轮询 `GET /api/b/batch/{batch_id}`，更新「N/M」。
4. `done` → 刷新 `GET /api/videos?type=viral&batch_id=...`，裂变陈列面出现 30~50 条（0 元标签）。

### 6.4 反馈 → 候选池（super_admin 审核）
1. 裂变卡片点「反馈」→ good/bad + 标签 + 心得 → `POST /api/videos/{id}/feedback` → 生成 `candidate(pending)`。
2. super_admin 在候选池看到 → 审核通过 / 驳回（P1 仅置状态，不推送阿里云大库）。

## 7. 轮询与节奏建议
- A台任务：`GET /api/tasks/{id}` 每 2s 一次，`done/failed` 停止，最长 5 分钟超时提示。
- B台批次：`GET /api/b/batch/{batch_id}` 每 1.5s 一次，`done/failed` 停止。
- 列表刷新：任务/批次完成事件触发一次刷新即可，不做常驻轮询。
- 埋点：fire-and-forget，无需等待响应，失败不重试不提示。
