# V1.3A.9 R5→R6 差异说明

## 核心变更

### P0-1: 执行完整性门禁
- 新增 `build_release.py`: 单一构建入口, 按顺序执行 truth gate → mutation → fault injection → 生成报告 → manifest → 验证
- 新增 `change_evidence.json`: 记录必改文件R5/R6 SHA, 任一未变化则exit 1

### P0-2: restore原子回滚修复
- `vault_recovery_service.py` 重写:
  - rollback point在关闭旧连接前创建 (checkpoint+fsync)
  - rollback point创建失败时旧连接不关闭
  - 回滚后强制比较canonical SHA和全表行数
  - 候选文件用tempfile.mkstemp
  - 禁止裸except

### P0-3: 24项验收重建
- `run_truth_gate.py` 完全重写vault section, 按工单§4的1-24编号:
  - 6: expires_at-1有效
  - 7: expires_at过期
  - 9: disabled token
  - 10: left token
  - 11: role downgrade
  - 12: store drift
  - 16: owner same-store rotate success
  - 17: cross-store read/write/rotate
  - 18: system→recovery转换
  - 19/20: 双向签名根隔离
  - 21: recovery token过期
  - 22: missing platform key exit 2
  - 23: restore成功完整证明(repo+ext一致, FK=1, integrity=ok)
  - 24: restore失败不变量

### P0-4: mutation框架执法
- `run_mutation_tests.py` 重写:
  - expected_hits参与判定: actual≠expected → ERROR → exit 1
  - BLOCKED要求: hits_match + SHA_changed + SHA_valid + exit 1
  - m10: verifier mix (recovery_provider.verify → repo.provider.verify)
  - m11: duplicate reconnect count
  - m12: unique tautology assertion

### P0-5: mutation_report闭合
- `--write-report`写入ORIGIN
- 默认模式: 缺文件→ERROR; 字节比较

### P1-1: 资源卫生
- 删除所有裸except (改为except OSError/FileNotFoundError)
- 候选文件用tempfile.mkstemp+fsync
- rollback文件用fsync

### 新增文件
- `build_release.py`
- `change_evidence.json`
- `restore_fault_injection_report.json`
- `tests/test_restore_fault_injection.py`
- `V1.3A.9_R5_TO_R6_DIFF.md`
