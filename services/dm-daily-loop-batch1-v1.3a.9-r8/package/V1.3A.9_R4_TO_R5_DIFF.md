# V1.3A.9 R4→R5 差异说明

## 核心变更

### P0-1: restore原子回滚
- 重写`VaultRecoveryService`为三阶段: prepare→commit→rollback
- prepare: 唯一候选文件(tempfile.mkstemp)→fsync→integrity→schema→SHA记录→不触碰原Vault
- commit: rollback point(复制原库+fsync) → 关闭旧连接 → os.replace → 验证新库 → 回写Repository
- commit失败: 用rollback point恢复原库 → 重连 → 验证原SHA/行数不变
- 删除所有裸except

### P0-2: mutation框架执法
- expected_hits参与判定: actual≠expected → ERROR → runner exit 1
- 默认模式: 缺mutation_report.json → ERROR (非WARNING)
- --write-report模式: 写入ORIGIN

### P0-3: 24项验收重建
- 按工单§4的1-24编号逐项实现
- security_invariants_report.json键名1_...到24_...

### P0-4: mutation_report闭合
- --write-report写入ORIGIN+纳入Manifest
- 默认模式字节比较

### P1-1: 资源卫生
- 删除所有裸except

- rollback文件用fsync

### P1-2: 报告真实
