# CLAUDE.md — 协作与发布执行纪律（项目固化）

> 本文件固化《美业 AI 项目代码协作与发布执行 SOP（固化版）》的执行纪律。
> **自即刻起为默认纪律，除非吴哥明确改令，一律按此执行。**
> 任何人/任何 Agent 在本仓库工作，都必须遵守门禁链路与 Git 规则。

## 门禁链路（不得跳步）
```
Claude 开发 → 推 Git 开发/评审分支
→ Qoder 精装修 → 推 Git 精装修/候选分支
→ 扣子组织终审（扣子 + 爱马仕 + GPT Coder）
→ 终审通过 → Qoder 版本进入 main / release
→ 扣子发布后端到阿里云 → 前端人工部署
```
任何代码不得绕过 Git 主仓库直接上线。

## 角色一句话
Claude = 写 · Qoder = 成 · 扣子 = 审 + 发 · 爱马仕 = 验 · GPT Coder = 终审把关 · 吴哥 = 拍板 · ChatGPT = 总协调/总把关。

## Claude 在本仓库的硬约束
**只做**：代码开发/修复/升级/方案落地，按工单边界。
**必交付**：`changed files` / `git diff --stat` / 测试结果 / 审查包 / 变更说明。
**提交去向**：只推**开发/评审分支**，命名 `review/<ticket>-claude` 或 `feature/<ticket>-claude`。

**红线（禁止）**
- 不直接推 `main`
- 不直接发布 / 不直接部署生产或测试环境
- 不绕过 Qoder 精装修；不绕过扣子终审
- 不擅自修改发布流程
- 不把密钥/token 明文写入代码库（真实密钥只走环境变量/密文注入）
- 未经审批不改 Nginx / SSL / 安全组 / 生产库

## Git 规则（强制）
1. 所有开发成果必须回到本 Git 仓库；不得只停留在本地目录 / 云端临时目录 / 聊天附件 / 打包文件 / 云助手上传文件。
2. `main` 只进"Qoder 精装修后 + 扣子终审通过"的版本；**Claude 原始开发版不直接进 main**。
3. 发布源只能是 `main` / `release tag` / 明确 `RC commit`；不得用未入库代码、服务器手改、聊天 patch、临时 zip。

## 安全与可追溯
- 真实密钥只走环境变量/密文注入。
- 所有版本有来源、有提交号、有审查记录、有发布记录；所有上线动作可追溯到具体 commit。
