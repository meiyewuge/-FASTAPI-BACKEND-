# WECHAT_MINIPROGRAM_DECISION.md｜是否需要改成微信小程序？

## 结论

MVP V0.1 暂时不建议直接改成微信小程序原生 + 云开发。

原因：

1. 当前系统有后端规则评分、AI报告、PDF生成、后台管理，H5 + 自有后端更适合快速跑通。
2. 微信云开发适合轻量小程序，但 PDF生成、AI报告、后台线索管理、后期多端扩展，用自有后端更稳。
3. 用户当前目标是让技术小伙伴或 Qoder 把服务拉起来，让用户能用；H5链接最快。
4. 微信小程序可以作为 V0.2 或 V1.0 的新增前端，不要替换当前后端。

## 推荐路线

### 第一步：先跑通 H5 版

让用户通过链接填写诊断和月度体检。

### 第二步：稳定后新增小程序前端

新增目录：

```text
mini-program/
```

小程序只负责前端展示和表单提交，后端仍调用现有 API。

### 第三步：再考虑微信登录、订阅消息、支付

这些属于后续商业增强功能。
